Data Ingestion from Rdbms to Hadoop/Hive.

ETL Control interface is created using Django 2.3, Python 3.6, Bootstrap, jquery, css other script languages. It is used to manage the data ingestion from Teradata, Oracle, SQL server, and DB2 to Hadoop. It can also load flat files, like Excel file, csv file, or other delimited text files. The ingested data from varieties of data sources makes the data center in Hadoop for RLTG Reporting Platform from which the Tableau reports get data. The ETL Control manages the data pipe line and control the flow execution. It significantly simplifies the complex, lengthy and inefficient ETL process.

Oracle Stored Proc + Sqoop + Django + Python = Etl Control

Oracle Stored Proc and Sqoop, is the major batch processing tools, Django 2.3 and Python 3 is used to create the Etl Control front tool to control the Execution of batch process, Monitor the current running process and manage user accessing the Etl Control.

This front end was developed to monitor the execution of stored proc to ingest the data from oracle to hadoop.
